# Share V1 Closeout

Date: 2026-05-16

Scope: documentation closeout for the current enterprise Share V1 state. This
report records boundaries and deferred items only; it does not add Share
features or alter public/protected behavior.

## Completed Capabilities

- Document public share.
- Collection public share.
- Library public share behind `Permissions:PublicShareLinks:AllowLibraryScope`.
- Workspace public scope remains unsupported.
- Public-safe tree projection for collection/library public links.
- Scoped readonly document reads inside public collection/library scope.
- Public Knowledge Base reader V1 using `#public/share-links/{token}`.
- Container-context creation entries:
  - document links from the document Share Drawer;
  - collection/folder links from container context;
  - library links from library header context.
- Document Share Drawer is document-only for creation and destructive
  management; broader collection/library links are read-only context there.
- Access & Sharing is the central governance surface for all document,
  collection, and library scope links.
- Audited copy for existing links through the protected copy endpoint.
- Pause, resume, and revoke management.
- Password and expiry policy enforcement for public links.
- Scope allow/deny policy for document, collection, and library public links.
- Governance analytics categories:
  - `resolve`
  - `document_view`
  - `tree_view`
  - `scope_denied`
  - `password_failed`
- Risk labels and token-safe reason summaries.
- Content protection metadata:
  - `disableDownload`
  - `disablePrint`
  - `disableCopy`
  - `watermarkEnabled`
  - token-safe `watermarkText`
- Public/protected API isolation.

## Security Boundaries

- Public reader does not enter `#editor` or the authenticated editor shell.
- Public reader uses anonymous public endpoints only:
  - `GET /api/v1/public/share-links/{token}/resolve`
  - `GET /api/v1/public/share-links/{token}/document`
  - `GET /api/v1/public/share-links/{token}/collection`
  - `GET /api/v1/public/share-links/{token}/tree`
  - `GET /api/v1/public/share-links/{token}/documents/{documentId}`
- Protected APIs are not widened by public links.
- Generic policy patch still rejects direct `linkMode = public`.
- Public share-link creation may set public policy internally only through the
  dedicated share-link workflow.
- Raw token, token hash, password, password hash, and password proof material
  must not be exposed in list/detail/audit/stat/public DTOs.
- Public links do not enable public edit, public comment, public download,
  public file download, public search, SEO, custom domains, themes, or public
  site behavior.
- Workspace public scope remains unsupported.
- Content protection is governance and deterrence, not DRM.
